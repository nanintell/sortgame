Merge Sort


The algorithm uses Divide and Conquer Strategy and its Big-O is O(n log(n)) for all cases.
Merge Sort's Best Case for ascending order is ascending sorted data while Worst Case is no duplicate data.
Merge Sort is done by halving data repeatedly until it can no longer being cut or only have a value left. Done then what?
Merge! but it's not just putting groups together. We check each value before put them in.
First, we check first position of each group first. The less value is winner and put into merged group first. Then, shift the won group to check the next position with the lost group. Again, less value is put into merged group.
Repeat this until all one of 2 groups' values are put into merged group then, throw in remaining values into merged group.
This merge method seems like sorting method but it may not give sorted data.
However, if we divide data into groups then use this method to merge them back, we get sorted value.

Now, let's see an example (read from left to right).
(Ps. In Battle Quest, you can use Divide function to halve data into groups and Merge function to merge group. Also, you need to sort the left group first then sort right group)

First Step Halve data until it can no longer do so.

Second Step Merge Data
When merging 2 and 4: 2 is less than 4 so put 2 in the group then the group 2 doesn't have member anymore then put all members in group 4 into the group
When merging 6 and 0: 0 is less than 6 so put 0 in the group then the group 0 doesn't have member anymore then put all members in group 6 into the group
When merging 2,4 and 0,6:
0 is less than 2 so put 0 into the group
shift checking position in group 0,6 to 6 then compare 2 with 6
2 is less than 6 so put 2 into the group
shift checking position in group 2,4 to 4 then compare 4 with 6
4 is less than 6 so put 4 into the group
then group 2,4 no longer has any members remained so put all remaining members in group 0,6 into the group Once done, we get the sorted data.
