Quick Sort


The algorithm uses Divide and Conquer Strategy and its Big-O is O(n log(n)) for best and average cases. For Worst Case, Big-O becomes O(n2)
Quick Sort's Best Case is data which gives estimately equal size from partitioning while Worst Case is sorted data.
This sorts uses Partitioning to sort values. Partitioning is a method where it selects a value called Pivot then rearrange data so that data on left side of Pivot is less or equal than it and data on right side of Pivot is more than it.
So what? How do we use it to sort? After doing Partition, we find that Pivot is already in a correct position of sorted data. However, the left and right of Pivot is not sorted yet. Thus, Quick Sort can be done by partitioning where Pivot is the first value.
We can see that it's like data is split into 3 parts, group with values less than Pivot, Pivot and group with values more than Pivot. After that, we keep partitioning data on each group to split again and again until we get a group of a value. We will get sorted data in asecending order (the least on left and most on right)

Now, let's see an example (read from left to right)
(Ps. In Battle Quest, Partition Function can be used by selecting range then Pivot. When you click "Done", the data will be partitioned. Also, for doing Quick Sort, you need to sort the left side first then on to right side)

First Step Partitioning whole data where Pivot is the first value.

Second Step Partitioning data on left side. We have to keep partitioning until the group is sorted before moving on.

Last Step Partitioning data on right side. Once done, we get sorted data in ascending order.

Ps. Partition used in this is Lomuto's partition which works by swapping Pivot with the last value
then use i to count values which is less than or equal to Pivot where i starts at first position - 1
After that, loop checking each data, if a value is less than or equal to Pivot, increment i by 1 and swap i with checking value (so that value is on the left side of Pivot)
Once finish looping, swap Pivot to position i + 1
Aside from Lomuto's, it also has Haore's which works by finding position of value on left side which is more than Pivot and value on right side which is less than Pivot. If found values and left position index is less than right position index, swap those values then keep checking until left and right side hit each other. Here is a video of the whole process.
