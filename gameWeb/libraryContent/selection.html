Selection Sort


The algorithm uses Decrease and Conquer Strategy and its Big-O is O(n2) for all cases
Selection Sort's Best Case in ascending order is ascending sorted data while Worst Case is descending sorted data.
Selection Sort in ascending order (left is the lowest and right is the highest) is done by "selecting" the lowest value and swap to the leftmost position.
Yeah Yeah. The least value is on the leftmost. Now what?
Why not find the second lowest value and place it in second position?
We can do it by finding the lowest value from second position to the last position since the lowest is already in the first position. Then Voila! First 2 values are sorted! Now what?
What will happen if we keep finding the next lowest and put them in the next position and keep at it until we find all of values?
We got sorted values!

Now let's see an example! (read from left to right) (Ps. In Battle Quest, You can use Find Min Function to get the lowest object in selected range.)

First Loop Find the lowest value in the group which is 2 then swap it with the first position.

Second Loop Find the lowest value from second to last position which is 4 then swap 4 with the second position.

Third Loop Find the lowest value from third to last position which is 5 but 5 is already at third position so no need to swap.

Last Loop Since all values are in right places, we can skip checking the last value. Once done, we get ascending sorted data.
